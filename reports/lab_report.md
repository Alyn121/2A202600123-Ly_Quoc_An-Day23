## 🔍 Edge Case & Failure Resilience

### 1. Transient Failures & Retry Bounding
The `evaluate` node continuously inspects tool outputs for error signatures. Upon detection, the state shifts to the `retry` node, incrementing the attempt counter. The graph's conditional edge gracefully short-circuits to the `dead_letter` queue when `max_attempts` is breached, guaranteeing immunity to infinite loops.

### 2. High-Risk Action Mitigation
Any query flagged with destructive keywords (e.g., "delete", "refund") triggers the `risky_action` state. Crucially, the payload is formulated but **halted**. Execution is fully gated behind the `approval` node. Rejections organically route to `clarify` rather than forcing an abort, preserving the user experience.
